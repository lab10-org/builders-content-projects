# /commit

Crea un commit enfocado del trabajo actual.

1. Verifica primero: `npm run typecheck` y `npm test` deben pasar.
2. Revisa los cambios (`git status`, `git diff`).
3. Agrupa solo los cambios relacionados con la tarea.
4. Mensaje en imperativo, una línea clara (ej: "Add lead scoring por ICP").
5. No incluyas cambios fuera de alcance.
